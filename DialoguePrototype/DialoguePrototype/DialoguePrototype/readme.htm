SunBurn Starter Kit

The SunBurn starter kit provides an empty solution for creating new SunBurn based games and projects.

To use:

  • Run this solution from Visual Studio (press F5 in Visual Studio)
  • When running press F11 to open the in-game editor
  • Import new models into the content repository (using the Scene Object tab)
  • Drag models from the repository into the scene tree-view
  • Add lights, adjust materials, the environment, and more!

Please note: this project uses forward rendering. To switch between forward and deferred rendering change the "renderingsystemtype" parameter in SceneInterface.CreateDefaultManagers() and the content processor used by the project's content repository (*.repo).

Also see the full SunBurn documentation.

Adding More Platforms

The starter kit initially creates a Windows game project. This is because models, textures, and other resources can only be imported while running a Windows version of the game (more details).

However adding additional platforms to the solution is very easy:

  • Right-click on the Windows project in Visual Studio's Solution Explorer
  • Select Create copy of Project for... (choosing the desired platform)
  • Open the new project's References
  • Select the "SynapseGaming-SunBurn" assembly and view its properties
  • Make sure the assembly's Copy Local property is set to True (required for both Xbox and WP7)
  • Select the "SynapseGaming-SunBurn-Physics" assembly and view its properties
  • Make sure the assembly's Copy Local property is set to True (required for both Xbox and WP7)

Both the Xbox 360 and Windows Phone 7 versions of a game can be created this way.

Generally no code or asset changes are necessary. However Windows Phone 7 only supports light mapped lighting (more details), so some tweaking may be necessary.

Please note: only the SunBurn Indie, Pro, and Studio editions support the Xbox and WP7 platforms.

SunBurn Class Quick-Reference

The following SunBurn classes are used to submit objects and lights for rendering, clear the scene, and other scene-management operations (view the full SunBurn API documentation):

SunBurnCoreSystem - required, core object for managing all engine resources.

ContentRepository - stores all assets imported via the SunBurn editor, this must be loaded from the XNA content pipeline before any other assets.

FrameBuffers - stores cached render targets (buffers) of a specific size. Can be used by several rendering passes, effectively sharing graphics resources between the passes. Multiple instances can be created each sized to a specific output target size (for instance to fit various sized viewports).

SceneInterface - service / component container that provides access to all scene managers, and allows controlling all managers from a single object.

SceneState - provides scene / view specific information to the lighting system.

RenderManager - provides full object, lighting, and shadow rendering support for scenes, XNA Models, and custom objects.

ObjectManager - scenegraph containing scene objects.

LightManager - scenegraph containing scene lights.

ShadowMapManager - shadow map cache and shadow optimization manager.

SunBurnEditor - allows the SunBurn Editor to connect with the engine, providing in-game editing.
